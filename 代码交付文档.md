# 债权图片清洗代码交付文档

## 1. 交付目标

本代码用于复用债权公告图片清洗流程：输入图片目录，调用视觉模型抽取债权字段，生成 CSV、检查报告，并可将审核前结果写入 MySQL staging 表。

本交付包只包含代码、配置样例和说明文档，不包含原始图片、抽取结果 CSV、模型缓存、数据库密码或 API Key。

## 2. 目录结构

| 路径 | 说明 |
|---|---|
| `extract_credit_claims.py` | 核心抽取脚本，支持整图模式、模型调用、字段清洗、CSV 输出 |
| `credit_claim_skill.py` | 工作流封装入口，提供 `prepare/extract/audit/load-db` 命令 |
| `audit_credit_claims.py` | 随机抽检报告生成脚本 |
| `verify_credit_claim_results.py` | 逐图逐条核对脚本 |
| `load_credit_claims_to_mysql.py` | MySQL staging 建表、字段注释和入库脚本 |
| `requirements.txt` | Python 依赖 |
| `.env.example` | 环境变量模板，不含真实密钥 |
| `docs/字段说明.csv` | 12 个业务字段和数据库字段映射 |
| `docs/人工修正记录.md` | 本次样例数据中的人工修正规则说明 |
| `skills/credit-claim-extraction/SKILL.md` | Codex skill 使用说明 |
| `新图片处理入库运行说明.md` | 面向本地运行的详细说明 |

## 4. 标准运行流程

```powershell
.\.venv\Scripts\python.exe .\credit_claim_skill.py prepare --input-dir .\image --output-dir .\outputs_image_whole
.\.venv\Scripts\python.exe .\credit_claim_skill.py smoke-test --input-dir .\image --output-dir .\outputs_image_whole_smoke --provider auto
.\.venv\Scripts\python.exe .\credit_claim_skill.py extract --input-dir .\image --output-dir .\outputs_image_whole --provider auto --smoke-first
.\.venv\Scripts\python.exe .\credit_claim_skill.py audit --output-dir .\outputs_image_whole --sample-size 8
.\.venv\Scripts\python.exe .\verify_credit_claim_results.py --output-dir .\outputs_image_whole
```

入库前先做连通和 dry-run：

```powershell
.\.venv\Scripts\python.exe .\credit_claim_skill.py load-db --check
.\.venv\Scripts\python.exe .\credit_claim_skill.py load-db --output-dir .\outputs_image_whole --dry-run
.\.venv\Scripts\python.exe .\credit_claim_skill.py load-db --output-dir .\outputs_image_whole --input-dir .\image
```

## 6. 数据库说明

脚本会创建或复用 3 张 staging 表：

- `credit_claim_runs`
- `credit_claim_records_staging`
- `credit_claim_issues`

表和字段均带 MySQL `COMMENT`，核心 12 个业务字段映射如下：

| 业务字段 | 数据库字段 |
|---|---|
| 债权人 | `creditor` |
| 主债务人 | `primary_debtor` |
| 借款金额 | `loan_amount` |
| 本金余额 | `principal_balance` |
| 保证人 | `guarantor` |
| 抵押物 | `mortgage` |
| 质押物 | `pledge` |
| 贷款日 | `loan_date` |
| 到期日 | `due_date` |
| 诉讼状态 | `litigation_status` |
| 受让方 | `assignee` |
| 转让方 | `transferor` |

## 7. 质量控制

质量控制分三层：

1. 规则检查：字段缺失、金额格式、日期格式、角色冲突、序号重复等。
2. 随机抽检：按来源图片覆盖抽样，生成抽检报告和预览图。
3. 逐条核对：按来源图片将抽取结果与原图再次交给视觉模型核对，输出逐条检查报告。

## 8. 已知口径

- 金额统一为阿拉伯数字加 `元`，原文为 `万元` 时换算为 `元`。
- 空字段保持为空，不从相邻行猜填。
- `保证人` 按业务口径兼容 `担保人名称/担保人/担保方`；若原文标注 `（抵押人）` 或 `（出质人）`，保留角色标注。
- `抵押物/质押物` 只放具体财产或权利，不放抵押人、出质人主体。

# 定时任务平台前端设计方案

## 目标

定时任务平台是本地采集系统的管理后台，用于查看采集运行状态、管理定时任务、追踪后台任务、检查队列和数据质量。前端继续采用单文件 Vue3 CDN 方案，避免引入 Vite、React 或复杂构建链。

本次页面定位为“后台工作台”，不是展示型首页。设计原则是紧凑、表格优先、状态明确、错误可追踪。

## 技术栈

| 模块 | 方案 |
| --- | --- |
| 后端 | FastAPI |
| 前端 | Vue 3 CDN + Element Plus |
| 图表 | ECharts CDN |
| 路由 | Hash 路由 |
| 数据源 | MySQL `auction_data` |

## 页面结构

| 页面 | 路由 | 作用 |
| --- | --- | --- |
| 运营总览 | `#/dashboard` | 查看调度器、后台任务、平台、批次和 AI 队列总览 |
| 任务管理 | `#/jobs` | 创建、编辑、启停、删除、立即运行采集任务 |
| 批次管理 | `#/batches` | 查看采集批次、状态、耗时和失败原因 |
| 标的查看 | `#/items` | 按平台、资产类型、项目状态、字段缺失筛选标的 |
| 标的详情 | `#/items/:id` | 查看单条标的字段、证据、资源和特有字段 |
| 队列管理 | `#/queues` | 查看 AI/OCR/附件解析任务状态，支持处理、重试、解锁 |
| 平台管理 | `#/platforms` | 按平台查看数据量、最近采集时间和成功率 |
| 质量报告 | `#/reports` | 查看采集质量报告、字段缺失和异常样本 |

## 功能闭环

### 任务运行状态

“立即运行”不再只返回成功提示，而是返回 `task_id`。前端通过以下接口轮询任务状态：

- `GET /api/tasks`
- `GET /api/tasks/{task_id}`

页面展示状态、耗时、错误日志和退出码，避免用户不知道任务是否还在运行。

### 调度器同步

任务新增、编辑、启停、删除后，后端会同步调度器状态，避免数据库记录和实际调度器任务不一致。

### Cron 校验

Cron 表达式在保存前由前端预览接口和后端保存接口双重校验。非法表达式会返回明确错误，不写入数据库。

### 错误处理

前端所有 API 请求统一处理错误，优先展示后端返回的 `detail` 或真实异常信息，而不是只显示“加载失败”。

## 视觉规范

- 浅色后台风格。
- 表格和筛选优先。
- 少装饰、少卡片、无 emoji 菜单。
- 状态统一使用 Element Plus 标签色。
- 空数据必须显示明确提示。

## 验证要求

每次修改后至少验证：

- 后端模块可编译。
- 核心 API 可访问。
- 首页、任务、批次、标的、队列、平台、报告页面可打开。
- 筛选、弹窗、表单校验、错误提示可用。
- 浏览器控制台无前端运行时错误。
